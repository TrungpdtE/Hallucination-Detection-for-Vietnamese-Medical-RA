# Lộ trình commit gợi ý cho GitHub

File này giúp chia project `rag-llm-vietnamese` thành nhiều commit nhỏ, có tiến trình phát triển rõ ràng như một project RAG/NLP được làm từng bước.

Repo này đã có sẵn nhiều file và data. Vì vậy cách làm hợp lý là commit theo từng nhóm chức năng, không commit một lần toàn bộ source.

## Nguyên tắc trước khi commit

Kiểm tra trạng thái repo:

```bash
git status
```

Không nên commit các file cache:

```text
__pycache__/
*.pyc
```

Nếu thấy cache Python đang xuất hiện, dọn bằng:

```bash
find src -name __pycache__ -type d -prune -exec rm -rf {} +
```

Nếu data quá lớn, cân nhắc chỉ commit sample/benchmark nhỏ và đưa dataset lớn lên Google Drive hoặc HuggingFace Hub.

## Kiểm tra trước khi push

Chạy nhanh các lệnh kiểm tra nhẹ:

```bash
python -m py_compile src/rag/hybrid_retriever.py
python -m py_compile src/eval/hallucination_baselines.py
python -m py_compile src/eval/generate_report.py
```

Nếu đã build vector store:

```bash
python -m src.rag.hybrid_retriever \
  --kb_dir data/kb_vi \
  --vectorstore outputs/vectorstore_vi \
  --question "Bệnh Hirschsprung là rối loạn đơn gen hay đa yếu tố?" \
  --top_k 5
```

## Push lên GitHub

```bash
git branch -M main
git remote add origin https://github.com/<your-username>/rag-llm-vietnamese.git
git push -u origin main
```

## Ghi chú về commit data lớn

Nếu GitHub báo file quá lớn:

```text
1. Không commit data.zip hoặc checkpoint model.
2. Đưa dataset/checkpoint lên HuggingFace Hub hoặc Google Drive.
3. README ghi rõ link tải data và cách đặt vào thư mục data/.
4. Commit code + sample nhỏ + benchmark/report là đủ đẹp cho CV.
```
